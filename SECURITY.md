# Security Policy

## Supported Versions

Security fixes are currently targeted at the latest `0.2.x` release line on
`main`.

| Version line | Supported |
| --- | --- |
| `0.2.x` | Yes |
| `< 0.2.0` | No |

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Instead, report it privately with:

- a concise description of the issue
- the affected version or commit
- reproduction steps or a proof of concept if available
- the potential impact
- any proposed mitigation or workaround you already identified

Use one of these private channels:

1. GitHub Security Advisories for this repository, if enabled
2. direct maintainer contact listed in the repository profile or release notes

## Response Expectations

The project aims to:

- acknowledge a credible report within `5` business days
- provide an initial triage outcome within `10` business days
- coordinate disclosure after a fix or mitigation is available

These are operational targets, not guaranteed SLAs.

## Scope Notes

- This policy covers repository code, release workflows, packaging, and the
  published documentation site.
- Scientific model quality issues, retrieval bias, or benchmark regressions are
  important, but they are not normally security vulnerabilities unless they
  create a concrete integrity, confidentiality, or availability risk.
